第二段：四条认证数据流

所有流程最终都只返回 accountId、服务端角色和独立会话；客户端不能自行声明账号或角色。

1 · 手机号验证码登录（首次即注册）
发送登录验证码
→
验证并消费验证码
→
查 phone_bindings
→
已有：恢复账号
没有：事务创建账号、手机号关系、member 用户
→ 签发设备会话 → 选择身份

整个流程不读取或写入微信绑定；当前 OPENID 只可用于短信风控辅助。

2 · 密码登录
输入手机号或账号名
→
唯一凭证关系解析 accountId
→
scrypt 校验账号密码
→
签发设备会话

手机号账号尚未设置密码时，不尝试创建新账号，提示“请使用验证码登录或先设置密码”。

3 · 微信登录入口
服务端读取可信 OPENID
→
已有微信绑定
恢复账号并直接签发会话
或
微信未绑定
→
进入手机号验证页
→
验证码成功，生成一次性手机号证明
→
弹窗询问是否绑定当前微信
暂不绑定
消费手机号证明 → 恢复或创建手机号账号 → 签发会话；微信关系零写入。
确认绑定
消费手机号证明 → 恢复或创建账号 → 事务确认双方均未绑定 → 建立微信关系 → 签发会话。

一次性手机号证明短期有效且只能消费一次，避免验证码通过后被重复用于创建会话或绑定。

4 · 账号与安全页主动绑定
绑定手机号
有效会话 → 发送“绑定用途”验证码 → 校验账号尚无手机号且号码未被占用 → 事务写入 phone_bindings 与账号反向引用。
绑定微信
有效会话 → 弹窗明确确认 → 服务端读取当前 OPENID → 校验双方均未绑定 → 事务写入 wechat_bindings 与账号反向引用。

首版只支持首次绑定；已有关系不允许覆盖、换绑或解绑。
