>更新时间：2026.06.09

外部App拉起微信客户端发起签约前，需先后台调用预签约接口完成预签约，获取pre\_entrustweb\_id，再拉起微信客户端，完成签约，返回App。

接口升级通知

委托代扣APP纯签约接口已于2025年9月23日升级为新版开发接口。此次升级采用平滑迁移策略：

- 存量模板：2025年9月23日前已申请APP纯签约的模板ID可继续使用原接口，旧接口暂不下线

- 新模板限制：2025年9月23日及之后申请的模板，以及此前未接入过APP签约的存量模板，将不再支持OpenBusinessWebview权限申请

- 新的调用方式：商户需使用WXLaunchMiniProgram Open SDK API调起微信签约


新版APP纯签约API文档：[APP调起签约](https://pay.weixin.qq.com/doc/v2/merchant/4015996790.md)



## 步骤1：预签约接口

### 接口说明

适用对象：直连商户

请求URL： https://api.mch.weixin.qq.com/papay/preentrustweb

请求方式： POST

数据格式： XML

签名方式： HMAC-SHA256、MD5

是否需要证书： 否

### 请求参数

| 参数名 | 变量 | 类型\[长度限制\] | 必填 | 描述 |
| --- | --- | --- | --- | --- |
| 应用ID | appid | string\[1,32\] | 是 | 直连商户申请的公众号或移动应用appid。 （请登录[开放平台](https://open.weixin.qq.com/)查看，注意与公众号的APPID不同）<br>示例值：wxcbda96de0b165486 |
| 商户号 | mch\_id | string\[1,32\] | 是 | 商户号是商户在微信申请微信支付成功后分配的账号ID，登录平台为pay.weixin.qq.com<br>示例值：1200009811 |
| 模板id | plan\_id | string\[1,28\] | 是 | 协议模板id，设置路径见[开发步骤](https://pay.weixin.qq.com/doc/v2/merchant/4011986709.md)。<br>示例值：12535 |
| 签约协议号 | contract\_code | string\[1,128\] | 是 | 商户侧的签约协议号，由商户生成，只能是数字、大小写字母的描述且在同一个商户号下唯一。<br>示例值：100000 |
| 请求序列号 | request\_serial | int 64 | 是 | 商户请求签约时的序列号，要求唯一性。禁止使用0开头，序列号主要用于排序，不作为查询条件，纯数字，范围不能超过int64的范围（9223372036854775807）。<br>示例值：1000 |
| 用户账户展示名称 | contract\_display\_account | string\[1,128\] | 是 | 签约用户的名称，用于用户签约页面的“开通账号”处进行展示，参数值不支持UTF8非3字节编码的字符，例如表情符号，所以请勿传微信昵称到该字段<br>示例值：微信代扣 |
| 回调通知url | notify\_url | string\[1,256\] | 是 | 用于接收签约成功消息的回调通知地址，以http或https开头，通知url必须为外网可访问的url，不能携带参数。 |
| 版本号 | version | string\[1,8\] | 是 | 固定值1.0<br>示例值：1.0 |
| 签名类型 | sign\_type | string\[1,32\] | 否 | MD5或者HMAC-SHA256，不填默认MD5<br>注意：app预签约的默认签名方式是(sign\_type为空或无该字段)：MD5<br>示例值：MD5 |
| 签名 | sign | string\[1,64\] | 是 | 详见[签名生成算法](https://pay.weixin.qq.com/doc/v2/merchant/4011985891.md)<br>示例值：C380BEC2BFD727A4B6845133519F3AD6 |
| 时间戳 | timestamp | string\[1,10\] | 是 | 系统当前时间，10位<br>示例值：1414488825 |
| 返回app | return\_app | string\[1,1\] | 否 | 用来控制签约页面结束后的返回路径（不传此参数，则签约完成后停留在微信内）。<br>Y表示返回app, 不填则不返回 注：签约参数appid必须为发起签约的app所有，且在微信开放平台注册过。<br>ios和android用户点完成按钮才返回原APP，直接右划关闭页面或左上角不支持返回APP<br>示例值：Y |


## 步骤2：签约接口

接入流程

一、该接口需要额外申请OpenBusinessWebview权限（仅支持在2025年9月22日之前申请过OpenBusinessView权限的商户在存量模板使用。2025年9月23日起，不再开放申请，且新增模板不可使用OpenBusinessView。请参考[APP调起签约-WXLaunchMiniProgram](https://pay.weixin.qq.com/doc/v2/merchant/4015996790.md)文档，使用WXLaunchMiniProgram，用户体验更好。），请联系对接您的运营同学协助申请，无法联系到运营同学时，可前往微信公众号「[腾讯客服](https://kf.qq.com/)」联系人工客服获取更多帮助，具体参看[接入流程](https://pay.weixin.qq.com/doc/v2/merchant/4011986709.md)

     邮件模板如下：

     标题：OpenBusinessWebview权限申请\_APP纯签约

     正文：

      （1）说明使用背景

      （2）appid、mchid（appid和mchid需要有绑定关系）

二、在微信开放平台（open.weixin.qq.com）注册APP信息

三、在开放平台-资源中心下载SDK：[资源下载](https://developers.weixin.qq.com/doc/oplatform/Downloads/iOS_Resource.html)

四、按照下面的接口指引文档完成对接

### 接口说明

适用对象：直连商户服务商

请求方式： POST

### 请求参数

| 参数名 | 变量 | 类型\[长度限制\] | 必填 | 描述 |
| --- | --- | --- | --- | --- |
| 预签约id | pre\_entrustweb\_id | string\[1,128\] | 是 | 预签约id<br>示例值：5778aadY9nltAsZzXixCkFIGYnV2V |

注意：

预签约id（pre\_entrustweb\_id）需要从[预签约接口](https://pay.weixin.qq.com/doc/v2/merchant/4011986804.md#%E6%AD%A5%E9%AA%A41%EF%BC%9A%E9%A2%84%E7%AD%BE%E7%BA%A6%E6%8E%A5%E5%8F%A3)的返回参数获取，其他参数请严格按照下方接入代码示例传入

### 接入代码示例：

IOS

```
WXOpenBusinessWebViewReq *req = [[WXOpenBusinessWebViewReq alloc] init];
req.businessType =12; //固定值
NSMutableDictionary *queryInfoDic = [NSMutableDictionary dictionary];
[queryInfoDic setObject:"5778aadY9nltAsZzXixCkFIGYnV2V" forKey:"pre_entrustweb_id"];
req.queryInfoDic = queryInfoDic;
[WxApi sendReq:req];
```

Android

```
WXOpenBusinessWebview.Req req = new WXOpenBusinessWebview.Req();
req.businessType = 12;//固定值
HashMap  queryInfo = new HashMap<>();
queryInfo.put("pre_entrustweb_id","5778aadY9nltAsZzXixCkFIGYnVV1");
req.queryInfo = queryInfo;
api.sendReq(req);
```

### 返回参数

WXOpenBusinessWebview.Resp，返回参数内容无需关注，如果签约成功，商户系统会收到“[签约结果通知](https://pay.weixin.qq.com/doc/v2/merchant/4011987586.md)”
