## 关于品牌API

为了在保证支付安全的前提下，带给品牌商户简单、一致且易用的开发体验，我们推出了全新的微信支付品牌API。

相较于现有的微信支付APIv3接口，主要区别是：

- 接口URL 命名规则由 `/v3/xxx` 变更为 `/brand/xxx`，其中/brand为固定前缀

- Authorization头中的 `WECHATPAY2-SHA256-RSA2048` 改为 `WECHATPAY-BRAND-SHA256-RSA2048`

- Authorization头中的 `mchid=xxx` 改为 `brand_id=xxx`

- 商户需要将微信支付[品牌公钥ID](https://pay.weixin.qq.com/doc/brand/4015415289.md)放入请求的HTTP头部 `Wechatpay-Serial` 中，用于验证公钥是否有效

- 只提供[公私钥模式](https://pay.weixin.qq.com/doc/brand/4015453439.md)，不再提供平台证书模式

- [微信支付公钥](https://pay.weixin.qq.com/doc/brand/4015453439.md)、[品牌API证书](https://pay.weixin.qq.com/doc/brand/4015415288.md)、[品牌API密钥](https://pay.weixin.qq.com/doc/brand/4015415287.md)都需要在[品牌经营平台](https://pay.weixin.qq.com/xdc/brandseccenter/index#/home/security-center)重新获取，与APIv3的微信支付公钥、API证书、API密钥不共用


注意事项：

开发联调阶段，调用接口后，建议在日志中保留应答的HTTP头 `Request-ID` 值， `Request-ID` 作为[请求的唯一标识](https://pay.weixin.qq.com/doc/brand/4015407525.md#%E8%AF%B7%E6%B1%82%E7%9A%84%E5%94%AF%E4%B8%80%E6%A0%87%E8%AF%86)，在调用接口遇到问题时，可向微信侧提供该值用于快速定位到请求记录，协助排查问题原因。
