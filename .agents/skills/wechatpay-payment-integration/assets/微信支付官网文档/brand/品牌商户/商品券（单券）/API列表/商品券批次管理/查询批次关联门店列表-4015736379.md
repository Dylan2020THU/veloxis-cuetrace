>更新时间：2025.08.28

品牌方可以通过该接口分页查询商品券批次所关联的门店列表

前置条件：

1. 批次的 `store_scope` 为 `SPECIFIC`

2. 批次未失效或过期





## 接口说明

支持商户：【品牌商户】

请求方式：【GET】 `/brand/marketing/product-coupon/product-coupons/{product_coupon_id}/stocks/{stock_id}/associated-stores`

请求域名：【主域名】https://api.mch.weixin.qq.com 使用该域名将访问就近的接入点

　　　　　【备域名】https://api2.mch.weixin.qq.com 使用该域名将访问异地的接入点，指引[点击查看](https://pay.weixin.qq.com/doc/brand/4015596635.md)

接口限频：20/秒（品牌ID维度）

## 请求参数

### Header 参数
| 参数名 | 类型 | 必填 | 描述 |
| --- |  --- | --- | --- |
| Authorization | string | true | 请参考 [签名认证](https://pay.weixin.qq.com/doc/brand/4015407568.md) 生成认证信息 |
| Accept | string | true | 请设置为 `application/json` |
| Wechatpay-Serial | string | true | 【微信支付公钥ID】  请传入brand\_id对应的微信支付公钥ID，接口将会校验两者的关联关系，参考 [微信支付公钥产品简介及使用说明](https://pay.weixin.qq.com/doc/brand/4015597001.md) 获取微信支付公钥ID和相关的介绍。以下两种场景将使用到微信支付公钥：<br>1、接收到接口的返回内容，需要使用微信支付公钥进行验签；<br>2、调用含有敏感信息参数（如姓名、身份证号码）的接口时，需要使用微信支付公钥加密敏感信息后再传输参数，加密指引请参考 [微信支付公钥加密敏感信息指引](https://pay.weixin.qq.com/doc/brand/4015407587.md)。 |
### Path 参数
| 参数名 | 类型 | 必填 | 描述 |
| --- |  --- | --- | --- |
| product_coupon_id | string | true | 【商品券ID】 商品券的唯一标识， [创建商品券](https://pay.weixin.qq.com/doc/brand/4015736297.md) 时由微信支付生成 |
| stock_id | string | true | 【批次ID】 商品券批次的唯一标识，商品券批次创建时由微信支付生成（可使用【 [创建商品券API](https://pay.weixin.qq.com/doc/brand/4015736297.md)】或【 [添加商品券批次API](https://pay.weixin.qq.com/doc/brand/4015736342.md)】创建），请确保该批次属于 `product_coupon_id` 对应的商品券 |
### Query 参数
| 参数名 | 类型 | 必填 | 描述 |
| --- |  --- | --- | --- |
| page_size | integer | false | 【分页大小】 单次拉取的数据条数上限，不填默认为20，最大值200 |
| page_token | string | false | 【分页Token】 分页查询时，需要传入上一次调用返回的 `next_page_token`，首次调用不填 |
## 应答参数

200 OK

| 参数名 | 类型 | 必填 | 描述 |
| --- |  --- | --- | --- |
| total_count | integer | false | 【总个数】 批次关联的门店总数，当且仅当 `page_token` 为空时提供 |
| store_list | array[object] | false | 【门店列表】 批次关联的门店列表 |
| store_list.store_id | string | true | 【门店ID】 门店的唯一标识，调用【 [创建品牌门店API](https://pay.weixin.qq.com/doc/brand/4015782988.md)】时由微信支付生成 |
| next_page_token | string(100) | false | 【下一页Token】 分页查询时，如果还有更多数据，会返回下一页的Token，请在下一次查询时设置在 `page_token` 中；如果已无更多数据，则不返回此字段。 |
