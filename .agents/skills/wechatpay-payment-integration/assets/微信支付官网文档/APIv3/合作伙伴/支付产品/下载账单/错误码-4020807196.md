## 二、错误码详细排查

### 270522881 — 请求的账单文件不存在

排查步骤：

1. 检查 `bill_date` 格式为 `YYYY-MM-DD`，且为昨天或更早的日期

2. 该日期当天若没有任何交易 / 资金流水，微信支付不会生成账单

3. 检查 `bill_type` 与商户实际开通的产品是否匹配


* * *

### 270522882 — 请求的账单正在生成中

排查步骤：

1. 账单一般在次日 9:00 之后生成，建议拉取时间放在 17:00 之后

2. 收到此错误后等待几分钟到 1 小时后重试，使用指数退避


* * *

### 269546505 — 商户证书已过期

排查步骤：

1. 登录商户平台 → 账户中心 → API 安全 → API 证书管理，查看证书状态

2. 已过期需重新申请新证书，下载并替换本地证书 / 私钥 / `serial_no`

3. 服务商场景下使用的是服务商商户号的证书


* * *

### 270522880 — 请求的账单日期已过期

排查步骤：

1. 微信支付仅保留近期账单（一般约 3 个月），超出范围的账单无法通过 API 下载

2. 业务侧应在账单生成后及时拉取并归档到自己的存储


* * *

### 268435472 — 请求过于频繁

排查步骤：

1. 同一商户号 / 同一账单日期的申请请求需控制频率，建议不超过 1 次/分钟

2. 收到限流后使用指数退避（1min / 3min / 10min）

3. 已拿到 `download_url` 后只下载一次即可，不要反复申请相同账单


* * *

### 270522884 — 账单日期格式不正确

排查步骤：

1. `bill_date` 必须为 `YYYY-MM-DD` 格式（如 `2025-08-01`），不能是 `20250801` / `2025/08/01` / `2025-8-1`

2. 月份和日期都必须是两位数，不足两位需补 0

3. 不要传入时间部分（如 `2025-08-01T00:00:00`）


* * *

### 269546525 — 错误的签名，导致验签失败

排查步骤：

1. 检查签名串拼接： `HTTP方法\nURL路径\n时间戳\n随机串\n请求体\n`（GET 请求 body 为空，末尾仍要 `\n`）

2. URL 路径要包含 query string，但不包含 host

3. `timestamp` 是秒级 Unix 时间戳（10 位），不是毫秒

4. 私钥用于 SHA256-RSA 签名，再 Base64 编码

5. 确认 `serial_no` 与私钥配对

6. 建议使用官方 SDK


* * *

### 269546507 — 商户证书序列号有误

排查步骤：

1. 登录商户平台 → 账户中心 → API 安全 → API 证书管理，确认当前证书序列号

2. `serial_no` 与签名所用私钥必须是同一对证书

3. 近期重新申请过证书的需更新本地配置

4. 服务商场景下使用的是服务商商户号的证书


* * *

### 269546538 — 商户号不存在

排查步骤：

1. 核对 `mchid` 与商户平台显示的商户号是否一致

2. 检查商户号是否被注销 / 关停

3. 新进件商户号尚未生效（一般几分钟到几小时）


* * *

### 270522890 — 不是指定的子商户的父商户

排查步骤：

1. 确认 `sub_mchid` 是否在当前服务商（ `sp_mchid`）下进件 / 绑定

2. 跨服务商的子商户无法互相拉账单


* * *

### 270924332 — 认证信息解析失败

排查步骤：

1. 检查 `Authorization` 头格式： `WECHATPAY2-SHA256-RSA2048 mchid="...",nonce_str="...",signature="...",timestamp="...",serial_no="..."`

2. 各字段用英文逗号分隔，字段值必须用英文双引号包裹

3. `signature` 必须是 Base64 编码后的字符串

4. 建议使用官方 SDK 自动生成 Authorization 头


* * *

### 268435463 — 参数过短

排查步骤：

1. 检查 `mchid` / `sub_mchid` 长度

2. 检查 `bill_date` 必须是完整的 10 位日期格式

3. 可选字段若传入则不能是空字符串


* * *

### 270522893 — 小微商户不单独提供对账单下载

排查步骤：

1. 小微商户没有独立账单，其交易需通过父商户（服务商）的账单查看

2. 如需查看个别小微商户的交易，可用「按时间段查询订单」接口逐笔获取


* * *

### 270522898 — 未开通下载二级商户资金账单产品权限

排查步骤：

1. 服务商平台 → 产品中心 → 「二级商户资金账单」，确认产品状态

2. 已开通仍报错则检查特约商户是否已授权该产品给服务商

3. 子商户商户平台 → 我的授权产品 → 授权后约 10 分钟生效


* * *

### 268435461 — 参数非法

排查步骤：

1. 检查 `bill_date` 是 `YYYY-MM-DD` 格式且不为未来日期

2. 检查 `bill_type` 是否在枚举范围（交易账单： `ALL` / `SUCCESS` / `REFUND`；资金账单： `BASIC` / `OPERATION` / `FEES`）

3. 检查 `tar_type` 是否为 `GZIP`（不传则不压缩）

4. 检查是否有不可见字符（BOM / 全角空格）


* * *

### 270924360 — 签名信息错误，验签失败

排查步骤：

1. 确认使用 SHA256-RSA2048 算法签名

2. 私钥参与签名（不要用证书公钥签名）

3. Base64 编码用标准 Base64，不是 URL-Safe Base64

4. `nonce_str` 为 32 位随机字符串

5. `timestamp` 为秒级 Unix 时间戳（10 位）

6. 建议使用官方 SDK


* * *

### 270522892 — 无效的商户号

排查步骤：

1. 检查 `mchid` / `sub_mchid` 为纯数字字符串，不带前缀或中横线

2. 商户号被关停 / 注销则无法恢复使用


* * *

### 270924333 — 时间戳距离当前时间超过 5 分钟

排查步骤：

1. 业务服务器开启 NTP 时间同步，确保系统时间准确

2. `timestamp` 必须是秒级 Unix 时间戳（10 位），不是毫秒（13 位）

3. 每次请求实时生成，不要用本地静态时间或缓存值


* * *

### 270522900 — 无权调用此接口

排查步骤：

1. 该接口仅对「电商收付通服务商」开放，普通服务商或直连商户不能调用

2. 直连商户应使用标准资金账单 / 交易账单接口（ `/v3/bill/fundflowbill` / `/v3/bill/tradebill`）


* * *

### 268435462 — 参数过长

排查步骤：

1. 检查 `mchid` / `sub_mchid` 长度

2. `bill_date` 严格为 10 个字符（ `YYYY-MM-DD`）

3. 对照接口文档逐字段核对最大长度
