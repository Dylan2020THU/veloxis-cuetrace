## 二、错误码详细排查

### 272757274 — 批次已过期

排查步骤：

1. 先调「查询批次详情」确认批次状态与时间窗口

2. 检查当前时间是否在 `[available_begin_time, available_end_time]` 区间内

3. 批次过期后无法续期，需创建新批次并把新 `stock_id` 用于建立委托关系

4. 检查 `business_type` 是否正确： `FAVOR_STOCK`（代金券） / `BUSIFAVOR_STOCK`（商家券）


* * *

### 269546538 — 商户号不存在

排查步骤：

1. 仔细核对商户号（10 位纯数字），前后不能有空格

2. 区分测试 / 灰度 / 生产环境的商户号

3. 检查请求头 `Authorization` 中签名时使用的 `mchid` 与代码中传递的 `mchid` 是否一致

4. 商户号被注销 / 冻结时需联系微信支付客服核实


* * *

### 270924345 — Content-Type 不正确

排查步骤：

1. 打印实际发送的请求头，确认 `Content-Type: application/json`

2. Body 必须是 JSON 字符串（如 `{"partner":{"type":"APPID",...}}`），不是表单字段

3. 同时确认 `Idempotency-Key` 头也带上了（建立合作关系接口必填）

4. 建议使用官方 SDK


* * *

### 272757265 — 批次非运营中

排查步骤：

1. 调「查询批次详情」确认 `status` 是否为 `running`

2. `unactivated` → 先调「激活代金券批次」接口

3. `paused` → 先调「重启代金券批次」接口

4. `stoped` 是终态，只能新建批次


* * *

### 272753677 — 批次归属方与 AppID 无绑定关系

排查步骤：

1. 登录批次创建方商户号 → 商户平台 → 账户中心 → API 安全 → AppID 账号管理

2. 把要授权的 `appid` 添加到该商户号的关联列表

3. 公众号 AppID、小程序 AppID、移动应用 AppID 需分别绑定

4. 服务商场景下 AppID 需绑定到子商户号上
