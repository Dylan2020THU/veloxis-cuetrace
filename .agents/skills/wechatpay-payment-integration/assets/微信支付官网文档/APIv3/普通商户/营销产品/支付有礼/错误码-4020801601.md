## 二、错误码详细排查

### 269546538 — 商户号不存在

排查步骤：

1. 仔细核对商户号（10 位纯数字），前后不能有空格

2. 业务侧入参校验： `^[0-9]{10}$`，不匹配直接拒绝

3. 区分测试 / 灰度 / 生产环境的商户号

4. 商户号被注销 / 冻结时需联系微信支付客服核实


* * *

### 270924345 — Content-Type 不正确

排查步骤：

1. 打印实际发送的请求头，确认 `Content-Type: application/json`

2. Body 必须是 JSON 字符串，不是表单字段

3. 多层代理 / 网关可能改写 Content-Type，做链路抓包定位

4. 建议使用官方 SDK，会自动设置正确的 Content-Type


* * *

### 272754695 — 请求参数基本规则错误

排查步骤：

1. 看响应体 `detail.field` 定位具体非法字段

2. 关键字段规则：

   - `activity_name` ≤ 10 字符

   - `activity_second_title` ≤ 9 字符

   - `merchant_logo_url` ≤ 128 字符，必须是 HTTPS URL（图片需先通过图片上传接口上传）

   - 活动有效期最长 90 天，且 `begin_time` 必须在创建日 1 年内

   - `available_day_time` 格式 `HHmmss`（如 `110000` 表示 11:00:00）

   - `delivery_purpose=JUMP_MINI_APP` 时 `mini_programs_appid` 和 `mini_programs_path` 都是必填
3. 时间字段必须包含时区（ `+08:00`）


* * *

### 269546505 — 商户证书已过期

排查步骤：

1. 登录商户平台 → 账户中心 → API 安全 → API 证书，查看到期时间

2. 过期需重新申请并下载新证书，更新到所有服务节点

3. 建议在证书过期前 30 天监控告警


* * *

### 270924346 — 签名信息非法

排查步骤：

1. `Authorization` 头必须以 `WECHATPAY2-SHA256-RSA2048 ` 开头（注意空格）

2. 必须包含五个字段： `mchid` / `nonce_str` / `signature` / `timestamp` / `serial_no`

3. 字段间用英文逗号分隔，字段值用英文双引号包裹

4. `signature` 用商户 API 证书私钥做 SHA256withRSA 签名，标准 Base64 编码（非 URL-Safe）

5. `nonce_str` 32 位随机字符串， `timestamp` 秒级 Unix 时间戳

6. 建议使用官方 SDK 自动生成 Authorization


* * *

### 270924291 — 头部信息不完整

排查步骤：

1. 必填请求头： `Authorization` / `Accept` / `User-Agent`，POST 请求还需 `Content-Type`

2. 容易遗漏 `User-Agent`——部分 HTTP 客户端默认不带，需显式设置

3. 多层代理 / 网关可能删除部分头，做链路抓包定位


* * *

### 270103055 — 修改活动状态请求过于频繁

排查步骤：

1. 同一个 `activity_id` 的状态变更操作应串行，不要并发

2. 调用终止后先查活动状态再做下一步，不要盲目重试

3. 重试加指数退避（10s / 30s / 1min）

4. 终态（ `TERMINATE_ACT_STATUS` / `OVER_TIME_ACT_STATUS`）下不能再变更状态


* * *

### 270080601 — 无效的活动 code

排查步骤：

1. `activity_id` 长度 ≤ 20 字符，由创建活动接口返回

2. 核对是否传错（被截断 / 编码污染 / 大小写）

3. 区分测试 / 生产环境的 `activity_id`，不要混用

4. 创建活动成功后立即把 `activity_id` 落库


* * *

### 268435461 — 参数非法

排查步骤：

1. 看响应体 `detail.field` 定位非法字段

2. 时间字段必须符合 RFC3339 格式： `yyyy-MM-DDTHH:mm:ss.sss+TIMEZONE`

3. 枚举值必须在白名单内（如 `delivery_purpose` 必须为 `OFF_LINE_PAY` 或 `JUMP_MINI_APP`）

4. `transaction_amount_minimum` 必须 \> 0

5. `available_day_time` 中 `begin_day_time` 不能 ≥ `end_day_time`

6. `begin_time` 不能 \> `end_time`，且活动跨度不超过 90 天


* * *

### 268455936 — 参数错误

排查步骤：

1. 首要：查看响应体 `message` 与 `detail`，里面会有具体说明

2. 创建活动前先确认：

   - 引用的商家券批次状态为运行中

   - 涉及的商户号都已开通对应产品权限

   - 投放小程序的 AppID 已与商户号绑定
3. 该错误码对应业务规则错误，需要根据 message 才能精确定位
