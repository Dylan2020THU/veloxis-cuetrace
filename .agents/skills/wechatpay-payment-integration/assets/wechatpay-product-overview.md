# 微信支付产品总览

以下按「商户 + 服务商通用」「服务商专属」两部分拆分。

---

## 通用产品（商户 & 服务商均支持）

### 支付产品

| 产品 | 产品概述 | API 版本 |
| --- | --- | --- |
| **JSAPI 支付** | 用户在微信内通过点击链接、扫码的方式打开商户自定义的H5页面并完成付款，适用于公众号商城、线下静态码、PC网站和自助终端场景 | V3 / V2 |
| **APP 支付** | 商户原生 APP 内收款（iOS / Android / 鸿蒙） | V3 / V2 |
| **H5 支付** | 用户使用手机浏览器打开商户自定义的H5页面，并在页面选购商品完成付款。不适用微信内打开H5页面、商户APP内打开H5页面场景 | V3 / V2 |
| **Native 支付** | 商户按微信支付的协议生成支付二维码，用户用微信扫码完成付款，适用于一单一码的场景 | V3 / V2 |
| **小程序支付** | 微信小程序内收款 | V3 / V2 |
| **JSAPI 合单支付** | 用户在微信内一次付款完成多个商户的合并下单，适用于公众号商城、线下静态码、PC网站等场景 | V3 |
| **APP 合单支付** | 用户在商户原生APP内一次付款完成多个商户的合并下单（iOS / Android / 鸿蒙） | V3 |
| **H5 合单支付** | 用户在手机浏览器内一次付款完成多个商户的合并下单，不适用微信内打开H5页面、商户APP内打开H5页面场景 | V3 |
| **Native 合单支付** | 商户按微信支付协议生成支付二维码，用户扫码一次付款完成多个商户的合并下单 | V3 |
| **小程序合单支付** | 用户在微信小程序内一次付款完成多个商户的合并下单 | V3 |
| **付款码支付** | 线下「被扫」面对面收银：用户出示付款码，商户扫码读取后发起扣款（商超 / 餐饮 / 自助机等） | V2 |
| **医保支付** | 用户激活微信医保电子凭证后线上完成挂号 / 门诊医保支付 | V3 |
| **订单退款** | 跨支付方式通用退款（申请 / 查询 / 异常退款 / 回调） | V3 |
| **下载账单** | 多维度对账文件（交易 / 资金 / 退款 / 分账） | V3 |
| **分账** | 商户将交易成功的资金按一定周期分给其他方（合作伙伴 / 员工 / 用户 / 分润方） | V3 / V2 |

以下是 6 种基础支付方式的详细选型对比：

| 维度 | JSAPI支付 | APP支付 | H5支付 | Native支付 | 小程序支付 | 付款码支付(V2) |
| --- | --- | --- | --- | --- | --- | --- |
| **用户支付环境** | 微信客户端内浏览器（PC网站/公众号/线下静态码展码，用户扫码后跳转微信内H5页面） | 原生APP（商户APP内直接调起微信支付） | 手机浏览器（非微信，通过链接跳转） | 微信客户端（PC网站展码，用户扫码后直接拉起支付，无网页） | 微信小程序（小程序内直接调起微信支付） | 线下收银台（商户扫用户付款码） |
| **典型场景** | PC网站展码扫码支付、公众号商城、线下静态码扫码后在微信内打开商户页支付（PC网站收银优选，能力覆盖Native且支持扫码后交互和用户识别） | 电商APP、外卖APP | 微信外手机浏览器购物 | PC网站/电商站极简扫码直付（扫码直接拉起付款，无中间页，无法识别用户身份，适合无需交互和会员识别的简单收款） | 微信小程序商城 | 超市/便利店/餐饮（被扫） |
| **用户扫码后交互能力** | ✅ 用户扫码后可展示商品选择、菜单、确认页等自定义H5内容 | ❌ | ❌ | ❌ 用户扫码后直接拉起付款，无中间交互页面 | ✅ 用户扫码进入小程序后可自定义交互页面再发起支付 | ❌（商户扫用户付款码） |
| **用户身份（openid）** | ✅ 有openid，可做会员识别、个性化推荐、用户画像 | ❌ | ❌ | ❌ 无openid，无法识别用户身份 | ✅（wx.login 获取） | ❌ |
| **调起方式** | WeixinJSBridge.invoke | OpenSDK sendReq | 浏览器跳转h5_url | 用户扫code_url二维码 | wx.requestPayment | 商户扫用户付款码直接扣款 |
| **下单产物** | prepay_id（2小时有效） | prepay_id（2小时有效） | h5_url（**5分钟**有效） | code_url（2小时有效） | prepay_id（2小时有效） | 无（一步扣款） |
| **应用类型要求** | 已认证服务号/政府或媒体类公众号 | 已认证开放平台账号 + 移动应用APPID | 已认证服务号/小程序/移动应用 | 已认证服务号/小程序/移动应用 | 已认证小程序 | 无特殊 |
| **前置配置** | JSAPI授权目录 | 开放平台移动应用APPID | H5支付域名 | 无特殊 | 小程序APPID | 无特殊 |
| **个体工商户** | ✅ | ✅ | ❌ | ✅ | ✅ | ✅ |
| **小微商户** | ✅ | ❌ | ❌ | ✅ | ✅ | ✅ |
| **推荐 API 版本** | V3（也有V2） | V3（也有V2） | V3（也有V2） | V3（也有V2） | V3（也有V2） | 仅V2 |

### 营销产品

| 产品 | 产品概述 | API 版本 |
| --- | --- | --- |
| **代金券** | 微信支付官方营销券（满减 / 折扣，支付前发券、支付中自动核销） | V3 |
| **委托营销** | 商户在商户平台创建券批次后授权小程序 AppID 发券（直播插件等场景） | V3 |
| **支付有礼** | 支付后营销：用户完成支付即自动发券 / 礼品，引导复购 | V3 |
| **小程序发券插件** | 小程序原生发券组件（免复杂开发即可发券） | V3 |
| **H5 发券** | H5 页面快速配置投放优惠券，缩短领券链路 | V3 |
| **智慧商圈** | 商业综合体 / 商圈数字化经营（跨店积分、支付即会员、商圈数据看板） | V3 |
| **支付即服务** | 支付完成页展示服务入口（售后 / 客服等场景快速连接） | V3 |
| **现金红包** | 商户通过公众号 / 服务通知向用户发放微信现金红包，用户领取后到账零钱，未领取 24 小时退回（拉新 / 促活 / 福利） | V2 |
| **委托代扣** | 用户一次性授权后，商户按周期自动扣费（订阅、月费、会员费等长期周期扣款场景） | V2 |

| **特约商户进件** | 服务商批量发起特约商户入网申请，替代手工录入（从业机构 / 电商平台不可用） | V3 |

| **商户被管控能力及原因查询** | 查询子商户支付 / 提现 / 退款 / 分账功能管控状态、原因与解脱路径 | V3 |
