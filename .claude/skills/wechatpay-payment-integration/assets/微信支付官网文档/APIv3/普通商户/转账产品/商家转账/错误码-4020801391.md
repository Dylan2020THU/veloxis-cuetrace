## 二、错误码详细排查

### 268530307 — 单据超过 30 天无法通过 API 查询

排查步骤：

1. 商家转账单据查询时效为 30 天，超过 30 天的转账单无法通过 API 查询

2. 业务侧应在 30 天内完成查询并落地存储关键字段

3. 历史明细可在商户平台 → 商家转账 → 转账记录在线查看


* * *

### 268529824 — 单号超出处理安全期

排查步骤：

1. 不要用很久之前生成的 `out_bill_no` 发起新的转账请求，新业务使用新的 `out_bill_no`

2. 收到此错误时先用查单接口确认原单状态，结果明确为失败时再换单重试


* * *

### 270924289 — 频率过快

排查步骤：

1. 单商户的发起转账接口频率上限为 100 次/秒

2. 业务侧应做请求级令牌桶限流

3. 收到限流后不要换单重试，使用相同 `out_bill_no` 降频重试（接口幂等）


* * *

### 268491071 — 账户余额不足

排查步骤：

1. 登录商户平台 → 资金管理 → 账户资金，查看「商家转账」账户余额

2. 余额不足时通过商户平台「充值」入金后再发起

3. 收到此错误时不要换单重试，使用相同 `out_bill_no` 在充值到账后原单重试


* * *

### 268495763 — 查询单号映射为空，订单不存在

排查步骤：

1. 核对 `out_bill_no` / `transfer_bill_no` 是否与发起时一致（大小写、长度）

2. 若发起请求超时但业务侧记录为"已发起"，可用同 `out_bill_no` \+ 原始参数原单重试

3. 业务侧应在发起前持久化 `out_bill_no` 与请求参数，保证可幂等重发


* * *

### 268479183 — 风控聚合问询收款账户异常

排查步骤：

1. 收款用户账户被风控（未实名 / 被冻结 / 收款受限等），无法接收转账

2. 引导用户在微信内完成实名认证（钱包 → 实名认证）

3. 用户已实名但仍报错，引导用户联系微信支付客服处理

4. 业务侧应暂停对该用户的后续转账，避免反复触发风控


* * *

### 268491067 — 此 IP 地址不允许调用该接口

排查步骤：

1. 登录商户平台 → 账户中心 → API 安全 → 接口安全 IP，确认是否启用了 IP 白名单

2. 把发起请求的服务器公网 IP 加入白名单（含 NAT 网关、负载均衡出口 IP）

3. 配置生效有几分钟延迟


* * *

### 268569882 — 请求频率过快

排查步骤：

1. 同一 `out_bill_no` 的重试不要小于秒级间隔，使用指数退避（1s / 3s / 10s）

2. 触发后不要换单，仍用同 `out_bill_no` 降频重试


* * *

### 268529837 — OpenID 和 AppID 不匹配

排查步骤：

1. OpenID 是 AppID 维度的，不同 AppID 下用户有不同 OpenID

2. 必须使用从相同 `appid` 转换得到的 `openid`

3. 业务侧持久化用户 OpenID 时也要记录所属 AppID


* * *

### 268435464 — 参数超出取值范围

排查步骤：

1. `transfer_amount` 单位为分，必须 \> 0，上限受场景设置与用户额度限制

2. `transfer_remark` ≤ 32 字符

3. `out_bill_no` ≤ 32 字符，只能是数字 + 大小写字母

4. `transfer_scene_id` ≤ 36 字符

5. `notify_url` ≤ 256 字符


* * *

### 268537402 — 单号超过查单有效期（不能撤销）

排查步骤：

1. 撤销仅适用于用户未确认收款前（ `ACCEPTED` / `PROCESSING` / `WAIT_USER_CONFIRM` 等状态）

2. 用户 24 小时未领取自动过期关闭后无法撤销

3. 终态单据（ `SUCCESS` / `FAIL` / `CANCELLED`）不能撤销


* * *

### 270924358 — 参数中含有非法的额外参数

排查步骤：

1. 移除请求体中接口文档未定义的字段

2. 不要把旧版商家转账（ `transfer-batches`）字段误传到新版（ `transfer-bills`）

3. 用官方 SDK 构建请求可避免字段拼写错误


* * *

### 268535970 — 当前状态不支持撤销

排查步骤：

1. 先调「商户单号查询转账单」接口确认当前 `state`

2. 仅以下状态可撤销： `ACCEPTED` / `PROCESSING` / `WAIT_USER_CONFIRM` / `TRANSFERING`

3. `SUCCESS`（已成功）资金已入用户零钱，无法撤销，需联系用户协商


* * *

### 268571344 — 授权非生效中，无法发起付款

排查步骤：

1. 该错误码出现在「免确认收款」模式下

2. 调「商户单号查询授权结果」接口确认授权 `state`，需为生效中（ `AUTHORIZED`）才能发起免确认转账

3. 授权已过期 / 已解除则需重新引导用户授权


* * *

### 269546507 — 商户证书序列号有误

排查步骤：

1. 登录商户平台 → 账户中心 → API 安全 → API 证书管理，确认当前生效证书序列号

2. `serial_no` 与签名所用私钥必须是同一对证书

3. 近期重新申请过证书的需更新本地配置


* * *

### 269546525 — 错误的签名，导致验签失败

排查步骤：

1. 检查签名串拼接： `HTTP方法\nURL路径\n时间戳\n随机串\n请求体\n`

2. URL 路径要包含 query string，但不能包含 host

3. `timestamp` 是秒级 Unix 时间戳， `nonce_str` 是 32 位随机字符串

4. 私钥用于 SHA256-RSA 签名，再 Base64 编码（标准 Base64）

5. `serial_no` 与私钥配对

6. 建议使用官方 SDK


* * *

### 268529835 — 单笔金额不在上下限范围

排查步骤：

1. 商户平台 → 产品中心 → 商家转账 → 转账场景，查看 / 调整各场景的金额上下限

2. 红包样式（ `user_recv_style.type=RED_PACKET`）单笔金额需 ≤ 200 元

3. 大额转账（≥ 2000 元）必须传 `user_name`（加密）


* * *

### 268491073 — MA 关系不存在

排查步骤：

1. 登录商户平台 → 账户中心 → API 安全 → AppID 账号管理

2. 把要使用的 `appid` 添加到商户号关联列表，等待绑定生效

3. 服务商场景下需绑定的是子商户与 AppID


* * *

### 268537840 — 单号不存在（不支持撤销）

排查步骤：

1. 核对 `out_bill_no` 是否与发起转账时完全一致

2. 先用商户单号查询接口确认原单是否存在

3. 查不到说明转账请求未真正受理，业务侧应排查发起接口的返回


* * *

### 268479986 — 商户命中不收不付合规管控-限制出资金类

排查步骤：

1. 登录商户平台 → 账户中心 → 消息 / 违约记录，查看具体管控类型与解除条件

2. 按指引申诉解除管控后等待几分钟到几小时生效

3. 服务商可使用「商户被管控能力及原因查询」接口获取详细信息
