## 二、错误码详细排查

### 271317510 — 记录不存在

排查步骤：

1. 检查 `out_order_no` 是否与创建时一致（大小写、长度）

2. 检查查询用的 `service_id` / `appid` 与创单时一致

3. 若创建请求曾因网络中断未真正成功，可用同 `out_order_no` 原参重新发起（接口支持原参重入）


* * *

### 271316574 — 支付分扣款失败

排查步骤：

1. 完结后微信支付会按一定频率自动重试扣款，不需要重新调用完结接口

2. 调用「查询支付分订单」接口查看 `collection.state` 判断进展

3. 长时间未扣成功可引导用户在「微信支付分小程序」查看订单并处理

4. 通过支付成功回调获取最终扣款结果


* * *

### 271302262 — 非法状态

排查步骤：

1. 先用「查询支付分订单」接口查当前 `state` \+ `state_description`

2. 状态机参考：

   - `CREATED` 已创建：可取消

   - `DOING` \+ `USER_CONFIRM`：可完结 / 取消 / 改金额

   - `DOING` \+ `MCH_COMPLETE`：等待自动扣款，不能再完结、不能取消

   - `DONE` / `REVOKED` / `EXPIRED`：终态，不能再操作

* * *

### 268435472 — 请求过于频繁

排查步骤：

1. 降低请求频率，使用指数退避策略（1s / 3s / 10s）

2. 同一 `out_order_no` 的请求需串行化处理


* * *

### 268435464 — 参数超出取值范围

排查步骤：

1. `out_order_no` ≤ 32 字符，字符集 `[0-9a-zA-Z_\-|]`

2. `service_id` 必须是 32 位数字

3. `service_introduction` ≤ 20 字符

4. `post_payments` 数组 ≤ 100 条； `post_discounts` 数组 ≤ 30 条

5. `risk_fund.amount` 必须 \> 0 且 ≤ 服务 ID 风险金额上限

6. `notify_url` ≤ 255 字符


* * *

### 271316598 — 重入真实服务结束时间不同

排查步骤：

1. 同一 `out_order_no` 重试时所有参数必须完全相同（包括时间字段）

2. 业务系统持久化首次请求的完整参数，重试时直接读取，不要重新生成时间


* * *

### 271316657 — 同步商户渠道成功，该订单状态不合法

排查步骤：

1. 同步订单状态接口（ `POST /v3/payscore/serviceorder/{out_order_no}/sync`）的前提是订单已走过完结流程

2. 正确顺序：用户确认 → 调用完结接口 → 订单进入 `MCH_COMPLETE` → 再调同步状态接口


* * *

### 271302144 — mchcode 无效

排查步骤：

1. `service_id` 必须是微信支付运营提供的 32 位数字 ID

2. 该 `service_id` 必须归属于当前 `mchid`

3. 多服务场景下不要混用不同服务的 `service_id`


* * *

### 271302149 — mchid 和 appuin 无绑定关系

排查步骤：

1. 登录商户平台 → 账户中心 → API 安全 → AppID 账号管理

2. 把要使用的 `appid` 添加到商户号关联列表

3. 创建订单的 `appid` 必须与完结 / 取消时使用的 `appid` 一致


* * *

### 271300099 — 分数不达标

排查步骤：

1. 用户支付分低于服务 ID 配置的最低准入分，无法创建订单

2. 引导用户在「微信支付分小程序」查看分数与提升建议

3. 业务侧建议在前端先调用「查询支付分」接口预判


* * *

### 271299627 — 当前有负面记录未处理，请处理后重试

排查步骤：

1. 引导用户在「微信支付分小程序」处理待处理事项后再创建新订单

2. 业务侧短期内不要对该用户反复重试


* * *

### 271316592 — 订单无法撤销

排查步骤：

1. 先调「查询支付分订单」确认状态

2. 允许撤销的状态： `CREATED`、 `DOING` \+ `USER_CONFIRM`、 `DOING` \+ `USER_PAYING`

3. 终态（ `DONE` / `REVOKED` / `EXPIRED`）不能撤销

4. 用户正在拉起收银台或自动扣款时撤销会失败，可等待 3 分钟后重试


* * *

### 270924332 — 认证信息解析失败

排查步骤：

1. 检查 `Authorization` 头格式： `WECHATPAY2-SHA256-RSA2048 mchid="...",nonce_str="...",signature="...",timestamp="...",serial_no="..."`

2. 各字段用英文逗号分隔，字段值用英文双引号包裹

3. `signature` 必须是标准 Base64

4. 建议使用官方 SDK 自动生成 Authorization 头


* * *

### 268503786 — 商户完结订单权限处于管控中

排查步骤：

1. 登录商户平台 → 账户中心 → 消息 / 违约记录，查看具体管控类型与解除条件

2. 按指引申诉处理后等待几分钟到几小时生效

3. 期间业务侧应暂停新创建订单，避免用户已确认订单无法完结


* * *

### 271316754 — 主动支付单交易过期时间未过期

排查步骤：

1. 用户正在主动支付页面，等用户主动支付流程结束（成功或自动过期）后再操作

2. 业务侧可轮询 `collection.state`，非 `USER_PAYING` 时再操作


* * *

### 50027158 — 客户交易限额（反洗钱等级较高风险客户）限制

排查步骤：

1. 用户被反洗钱风控判定为较高风险，商户侧无法绕过

2. 引导用户在微信内完成更高等级实名认证或联系微信支付客服


* * *

### 271316637 — 真实结束时间小于预计开始时间

排查步骤：

1. 完结订单时 `time_range.end_time` 必须晚于创单时的 `time_range.start_time`

2. 时间格式要求与首次保持一致（如都用 `yyyyMMddHHmmss`）

3. `end_time` 必须早于或等于调用完结接口的当前时间


* * *

### 271316656 — 总金额超过此服务的服务风险金额

排查步骤：

1. 先免模式： `total_amount` ≤ 创单时的 `risk_fund.amount` ≤ 服务 ID 风险金额上限

2. 先享模式： `total_amount` ≤ 服务 ID 风险金额上限

3. 创单时建议把 `risk_fund.amount` 设置为预估上限，避免实际结算超过押金

4. 如需提高上限请联系微信支付商务


* * *

### 268497610 — 待支付中的订单过多（uin 维度）

排查步骤：

1. 引导用户在「微信支付分小程序」处理已有的待支付订单

2. 业务侧不要对同一用户连续创建大量未完成的订单


* * *

### 271302333 — 商户发起催收时订单状态不合法

排查步骤：

1. 当前操作与订单状态不兼容（如订单已取消），先查询订单状态后按状态发起对应操作

2. 已取消订单是终态，如还需扣款需重新创建一笔订单
