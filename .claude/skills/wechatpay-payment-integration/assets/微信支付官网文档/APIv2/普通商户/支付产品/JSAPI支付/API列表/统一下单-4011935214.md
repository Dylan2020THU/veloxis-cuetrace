## 应用场景

除付款码支付场景以外，商户系统先调用该接口在微信支付服务后台生成预支付交易单，返回正确的预支付交易会话标识后再按Native、JSAPI、APP等不同场景生成交易串调起支付。

## 状态机

支付状态转变如下：

```mermaid
%%{init: {'themeVariables':{'edgeLabelBackground':'#ffffff'}}}%%
stateDiagram-v2
    direction LR
    state "NOTPAY(未支付)" as S1
    state "SUCCESS(支付成功)" as S2
    state "REFUND(已退款)" as S3
    state "CLOSED(已关单)" as S4
    state "PAYERROR(支付错误)" as S5

    [*] --> S1: 【下单】
    S1 --> S2: 【支付成功】
    S2 --> S3: 【发起退款】
    S1 --> S4: 【关单】
    S1 --> S5: 【支付错误】
    S3 --> [*]
    S4 --> [*]
    S5 --> [*]
```

## 错误码

| --- | --- | --- | --- |
| INVALID\_REQUEST | 参数错误 | 参数格式有误或者未按规则上传 | 订单重入时，要求参数值与原请求一致，请确认参数问题 |
| NOAUTH | 商户无此接口权限 | 商户未开通此接口权限 | 请商户前往申请此接口权限 |
