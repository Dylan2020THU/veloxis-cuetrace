# 账号先行登录流程设计

## 背景

当前登录页先选择身份，再进入账号登录或注册。用户确认需要改为：进入小程序后先进行账号登录/注册，账号验证成功后再选择本次进入的身份。

## 目标

1. 进入小程序时，首先展示账号登录/注册界面，不先展示身份选择。
2. 注册新账号时默认创建球员基础账号，即 `roles: ['member']`。
3. 账号登录、手机号登录、微信绑定登录通过后，再根据该账号的 `roles` 展示可选身份。
4. 用户选择身份后，再调用现有登录落地逻辑进入对应端口。
5. 教练账号可选球员/教练，店主账号可选店主，管理员 `admin_zhx` 可选球员/教练/店主。
6. 管理员选择店主身份时继续绕过店主资质审核；普通店主仍走资质审核网关。

## 非目标

1. 不新增身份类型。
2. 不新增账号体系或数据库结构。
3. 不改变现有账号规则、密码规则、短信验证码逻辑。
4. 不改变教练向下兼容球员、管理员唯一账号等既有规则。

## 页面流程

### 初始页面

初始进入登录页：

- 默认 `step = 'auth'`。
- 展示账号登录 / 手机号登录。
- 支持切换到注册。
- 支持微信登录绑定。
- 不展示球员、教练、店主身份卡片。

### 注册流程

注册表单不再依赖身份选择：

- 成功注册后保存账号为 `role: 'member'`、`roles: ['member']`。
- 注册成功后回到账号登录状态。

### 登录流程

账号密码或短信验证成功后：

- 不立即进入首页。
- 读取该账号可用 `roles`。
- 缓存待进入账号信息，例如账号名和角色列表。
- 切换到 `step = 'role'` 展示可选身份。

### 身份选择流程

身份选择页只展示当前账号有权限进入的身份：

- `member` 展示球员。
- `coach` 展示教练。
- `shop` 展示店主。

用户点击身份后：

- 调用 `doLogin(role, loginName, roles)`。
- 非店主走现有普通登录落地。
- 店主走现有店主资质审核网关。
- 管理员店主继续绕过资质审核。

## 数据规则

账号角色读取沿用当前本地账号结构：

- `roles` 优先。
- 旧账号没有 `roles` 时，按 `role` 迁移推导。
- 内置管理员账号读取时补齐 `roles: ['member', 'coach', 'shop']`。

新增本地页面状态：

- `pendingAccount`: 登录成功但尚未选择身份的账号名。
- `pendingRoles`: 登录成功账号的可用身份列表。

## 测试

需要覆盖：

1. 登录页初始状态不再是身份选择，而是账号登录。
2. 注册新账号默认保存为球员基础账号。
3. 账号密码验证成功后进入身份选择，不立即调用 `data.login`。
4. 身份选择页只显示账号可用身份。
5. 选择身份后调用 `data.login(role, roles, loginName)`。
6. 管理员账号登录后可选择三种身份。
7. 普通店主选择店主后仍走审核网关，管理员店主仍绕过审核。

## 验收标准

1. 打开小程序登录页，第一屏是账号登录/注册，不是身份选择。
2. 普通新注册账号登录后，只能选择球员。
3. 教练账号登录后，可选择球员或教练。
4. 管理员账号登录后，可选择球员、教练、店主。
5. 选择身份后进入对应端口。
6. 全量测试通过。
