# 管理员跨身份登录与个人页标识设计

## 背景

管理员账号用于管理平台能力，不应被建模为第四种业务身份。用户已确认：同一个管理员账号可以选择球员、教练、店主任一身份登录；若选择店主身份，应绕过店主资质审核直接进入店主端。

当前内置管理员账号为：

- 账号：admin_zhx
- 密码：2612694

## 目标

1. 管理员账号可以以球员、教练、店主任一身份登录。
2. 管理员选择店主身份登录时，绕过「店主资质审核」门槛，直接进入店主端。
3. 管理员账号不论以什么身份登录，「我的」页面头像下方均显示「管理员」。
4. 管理员账号不论以什么身份登录，「我的/设置」页面均显示「店主资质审核」功能。

## 非目标

1. 不新增第四种 `admin` 业务身份。
2. 不改变普通球员、教练、店主的登录规则。
3. 不新增管理员管理页面。
4. 不改变店主资质审核的审核流程，只调整管理员可见性和登录绕过逻辑。

## 设计

管理员状态作为独立权限叠加在当前业务身份之上：

- 当前登录身份仍由用户选择的 `member`、`coach`、`shop` 决定，用于控制底部栏、页面入口和业务数据视图。
- 是否管理员由现有管理员识别逻辑判断，用于控制管理功能可见性和个人页标签。
- `admin_zhx` 的可登录身份范围应覆盖 `member`、`coach`、`shop`。
- 如果本地已存在 `admin_zhx` 注册记录，应在读取注册账号时补齐管理员账号的三身份权限，避免旧缓存只允许球员端登录。

## 页面行为

### 登录页

管理员账号选择任一身份时：

- 账号密码校验仍使用内置管理员账号信息。
- 角色校验通过，不因当前身份不是注册身份而失败。
- 选择店主身份时，不跳转店主资质申请页，直接进入店主端首页。

普通账号保持现有逻辑：

- 未注册账号不能登录。
- 非店主账号不能直接进入店主端。
- 未通过店主审核的普通店主账号仍按原流程进入资质申请或等待状态。

### 我的页

管理员账号登录后：

- 昵称仍显示账号名或用户昵称。
- 头像下方角色文本固定显示「管理员」。
- 当前业务身份不在头像下方追加显示，避免误解为第四业务身份。

普通账号保持现有逻辑：

- 球员显示「会员」。
- 教练显示「教练」。
- 店主显示「店家」。

### 设置页

管理员账号登录后：

- 不论当前身份是球员、教练还是店主，都显示「店主资质审核」入口。

普通账号保持现有逻辑：

- 不显示「店主资质审核」入口。

## 测试

新增或扩展测试覆盖以下行为：

1. `admin_zhx` 的注册账号读取结果包含球员、教练、店主三种身份。
2. 管理员以店主身份登录时，跳过店主资质审核门槛。
3. 管理员以球员、教练、店主身份进入「我的」页时，头像下方均显示「管理员」。
4. 普通球员、教练、店主的「我的」页角色显示不变。
5. 管理员在任一身份的设置页都能看到「店主资质审核」入口，普通账号不可见。

## 验收标准

1. 使用 `admin_zhx / 2612694` 分别选择球员、教练、店主身份均可登录成功。
2. 选择店主身份时，管理员直接进入店主端，不进入资质审核申请页。
3. 三个身份的「我的」页头像下方均显示「管理员」。
4. 三个身份的「我的/设置」页面均显示「店主资质审核」。
5. 普通账号没有获得管理员入口或店主端绕过能力。
