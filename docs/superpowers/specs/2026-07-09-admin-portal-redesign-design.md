# 管理员端重构设计

## 背景

当前管理员能力叠加在球员、教练、店主三种身份之上。管理员登录后仍会进入身份选择流程，后续再以某个普通身份进入对应端侧。这会带来两个问题：

- 管理员权限和普通身份权限边界不清，容易出现普通账号被误识别为管理员的问题。
- 管理员需要查看平台级门店、教练、会员数据，但现有页面大多是店主或用户自己的数据视角。

本次重构将管理员作为独立端侧处理。管理员登录成功后不再选择身份，直接进入管理员首页。

## 目标

- 管理员账号输入正确账号密码后，跳过身份选择页，直接进入管理员端。
- 管理员端底栏只包含「门店」「教练」「会员」。
- 每个管理员页面都包含上方数据总览、筛选区、下方明细列表。
- 管理员端使用独立权限判断和独立聚合数据接口，不复用店主端的当前门店数据范围。
- 当前只有一个管理员账号：`admin_zhx`。密码继续沿用现有管理员账号配置，不在页面中单独展示或新增配置入口。

## 非目标

- 不新增后台管理 Web 系统。
- 不新增管理员创建、删除、改密页面。
- 不在本次加入封禁、导出、批量操作、运营配置、财务结算等高级后台能力。
- 不改变球员、教练、店主的正常登录和身份选择逻辑。

## 登录与权限

### 登录流程

```text
登录页输入账号密码
        ↓
本地注册账号校验
        ↓
判断账号是否为管理员账号
        ↓
管理员账号：校验管理员权限并进入管理员端
普通账号：进入身份选择页
```

管理员分流规则：

- 账号不是管理员账号：保持现有普通账号流程。
- 账号是管理员账号，但密码错误：提示密码错误，不进入管理员端。
- 账号是管理员账号且密码正确：调用管理员权限校验，校验通过后进入 `/pages/admin/stores/index`。

### 权限边界

管理员端不把管理员伪装为 `member`、`coach` 或 `shop`。客户端可以保留一个独立的管理员模式标记，例如 `adminMode` 或 storage 中的 `dc_role = 'admin'`，仅用于：

- 渲染管理员底栏。
- 控制管理员页面访问。
- 退出登录时清理管理员状态。

云函数读取平台全量数据前必须校验管理员权限，不能只依赖前端页面隐藏。

## 路由与底栏

新增管理员页面：

- `/pages/admin/stores/index`：门店
- `/pages/admin/coaches/index`：教练
- `/pages/admin/members/index`：会员

管理员底栏：

```text
门店 | 教练 | 会员
```

自定义底栏根据当前登录状态选择 tab 配置：

- `member`：沿用球员端底栏。
- `coach`：沿用教练端底栏。
- `shop`：沿用店主端底栏。
- `admin`：使用管理员底栏。

管理员页面顶部需要提供一个轻量入口用于退出登录。由于底栏不包含「我的」，退出入口放在页面右上角或页面顶部工具区。

## 数据接口

新增管理员专用数据接口，避免复用店主视角接口：

- `getAdminStores`
- `getAdminCoaches`
- `getAdminMembers`

三个接口都必须先校验当前用户是否为管理员。校验失败时返回无权限错误，前端提示后回到登录页。

### getAdminStores

数据来源：

- `stores`
- `shops`
- `shop_applications`
- 必要时读取 `users` 补充店主账号、昵称、订阅信息。

返回内容：

- `summary`
  - `totalStores`
  - `approvedStores`
  - `pendingApplications`
  - `rejectedApplications`
  - `checkinEnabledStores`
- `stores`
  - `storeId`
  - `storeName`
  - `ownerOpenid`
  - `ownerName`
  - `region`
  - `address`
  - `applicationStatus`
  - `checkinEnabled`
  - `createdAt`

支持筛选：

- 审核状态：全部、待审核、已通过、已驳回、未提交。
- 打卡状态：全部、已开启、未开启。
- 地区。
- 关键词：门店名、店主名、地址。

### getAdminCoaches

数据来源：

- `coaches`
- `users`
- `shop_coach_links`
- `coach_shop_applications`
- `stores`

返回内容：

- `summary`
  - `totalCoaches`
  - `boundCoaches`
  - `pendingApplications`
  - `unboundCoaches`
  - `activeCoaches`
- `coaches`
  - `coachOpenid`
  - `coachName`
  - `avatar`
  - `boundStoreName`
  - `bindingStatus`
  - `studentCount`
  - `createdAt`

支持筛选：

- 绑定状态：全部、已绑定、待审核、未绑定、已驳回。
- 所属门店。
- 关键词：教练昵称、账号、门店名。

### getAdminMembers

数据来源：

- `users`
- `members`
- `sessions`
- `coach_member_links`
- `coach_lessons`

返回内容：

- `summary`
  - `totalMembers`
  - `newToday`
  - `newThisWeek`
  - `trainedMembers`
  - `activeMembers`
- `members`
  - `memberOpenid`
  - `memberName`
  - `avatar`
  - `accountName`
  - `totalTrainingHours`
  - `trainingDays`
  - `lastTrainingAt`
  - `lastStoreName`
  - `createdAt`

支持筛选：

- 注册时间：全部、今天、本周、本月。
- 训练状态：全部、有训练记录、无训练记录。
- 最近训练门店。
- 关键词：昵称、账号。

## 页面设计

三个页面采用一致结构：

```text
页面标题 / 退出入口
数据总览卡片
筛选条
明细列表
空状态 / 加载态 / 错误态
```

### 门店页

顶部总览展示门店经营和审核状态，例如：

```text
门店总数 | 待审核 | 已通过 | 已开通打卡
```

列表以门店为主对象，每条展示：

- 门店名称
- 地址或地区
- 店主昵称或账号
- 审核状态
- 打卡开启状态
- 创建时间

店主资质审核归入门店页，不单独放入底栏。待审核门店可以从待审核统计卡片或列表条目进入现有审核页，审核能力仍只对管理员开放。

点击普通门店条目进入详情不是本次必做项；第一版先展示列表信息和待审核入口。

### 教练页

顶部总览展示教练绑定状态，例如：

```text
教练总数 | 已绑定 | 待审核 | 未绑定
```

列表以教练为主对象，每条展示：

- 头像和昵称
- 当前绑定门店
- 绑定状态
- 学员数
- 创建时间

### 会员页

顶部总览展示会员增长和训练活跃，例如：

```text
会员总数 | 今日新增 | 本周新增 | 有训练记录
```

列表以会员为主对象，每条展示：

- 头像和昵称
- 账号名
- 累计训练时长
- 训练天数
- 最近训练时间
- 最近训练门店

## 状态处理

- 加载中：显示页面级 loading 或列表 loading。
- 空数据：展示明确空状态，例如「暂无门店数据」。
- 无权限：提示「管理员权限已失效，请重新登录」，清理管理员状态并返回登录页。
- 接口失败：展示「数据加载失败，请稍后重试」，保留重试入口。

## 测试与验收

### 登录验收

- 管理员账号密码正确：直接进入「门店」页，不出现身份选择页。
- 管理员密码错误：提示密码错误。
- 普通账号登录：仍进入身份选择页。
- 普通账号不能访问 `/pages/admin/*` 页面。

### 底栏验收

- 管理员端底栏只显示「门店」「教练」「会员」。
- 三个 tab 可正常切换。
- 球员、教练、店主底栏不受影响。

### 数据验收

- 门店页能展示平台门店总览和门店明细。
- 教练页能展示平台教练总览和教练明细。
- 会员页能展示平台会员总览和会员明细。
- 筛选条件变更后，列表和总览保持一致。
- 无数据、接口失败、无权限状态有明确提示。

### 回归验收

- 球员登录、教练登录、店主登录流程保持可用。
- 店主资质审核入口仍只在管理员可访问位置出现，并从管理员「门店」页进入。
- 已有店主端「球厅」「教练」「会员」「球桌」页面不因管理员端新增页面而改变行为。
